What it is AND WHY YOU NEED IT

QUICK START (5 MINUTES)

You will need

  • Oracle Fusion reporting access (WSDL)
  • Java 8+
  • A JDBC client (DBeaver recommended)

Installation

Download the latest driver from GitHub Releases (ZIP/JAR) or build from source.

(Fusion setup) Import the sample OTBI artifacts: unarchive DM_ARB.xdm.catalog and RP_ARB.xdo.catalog from the repo's otbireport into your /Shared Folders/Custom/Financials (path can vary).

Configure your client

  • Register the driver JAR
  • Driver class: my.jdbc.wsdl_driver.WsdlDriver
  • JDBC URL (template):
jdbc:wsdl://<your-instance>.oraclecloud.com/xmlpserver/services/ExternalReportWSSService?WSDL:/Custom/Financials/RP_ARB.xdo
  • Use your Fusion username/password
  • Windows/DBeaver users: enable "view indexes" in driver settings if needed
  • Upgrades: for builds after Feb 27, 2025, delete local metadata.db and metadata.db.wal to refresh the cache

TO KEEP THINGS FAST:

  • Don't "Refresh" the root Indexes node (it triggers wide metadata scans)
  • Expand Tables â†’ your table â†’ Indexes to fetch only what you need
  • If you truly need all indexes, run a one-off SQL (e.g., ALL_INDEXES + ALL_IND_COLUMNS) instead of tree refresh
